## Input Data

## 分析プロセス

### Step 0. SERPs確認（必須）

分析に入る前に、ターゲットKWでWeb検索を実行し、上位10件の傾向を把握する。 以下を確認すること。

- 上位記事がどの検索意図に応えているか（情報収集 / 比較検討 / 手続き方法 / 体験談）
- 強調スニペットやPAAの有無と内容
- 上位を占めているサイトの種類（公的機関 / 企業 / 個人ブログ / Q&Aサイト）
- 記事形式の傾向（一覧 / ハウツー / 解説 / ランキング）

※この確認結果は出力しなくてよい。以降の分析精度を上げるためのインプットとして使う。

### Step 1. 検索者プロファイル

| 項目 | 分析内容 |
|---|---|
| 誰？ | 年齢層、職業、現在の状況（例：退職直後、転職活動中、育休中） |
| 検索の場面 | いつ・どこで・何がきっかけで検索するか（例：上司に怒られた直後にスマホで、確定申告の時期にPCで） |
| 知識レベル | このKWの周辺知識をどの程度持っているか。初心者 / 経験者 / 専門家 |
| 感情の温度 | 冷静に情報収集しているか、焦り・不安・怒りなどの感情を伴っているか |

### Step 2. 検索意図の分解

#### 意図の分岐判定

同じKWでも検索者の状況によって求める答えが異なる場合がある。 以下の基準で判定する。

- このKWを検索する人を2〜3パターン想定する
- 各パターンの「読了後に取る行動」が異なるかを確認する
- 行動が異なる → 意図が分岐している
- 行動が同じ → 分岐なし
- 分岐がある場合 → SERPs上位の傾向からメイン意図を特定し、サブ意図は「記事内で補足」か「別記事に分離」かを判断する

#### 顕在ニーズ

検索した瞬間に答えがほしいことを1文で定義する。 「〇〇な人が、□□を知りたい」の形で書く。

#### 潜在ニーズ

本人は言語化できていないが、教えると行動が変わる情報。以下の切り口で洗い出す。

- 知らないと損する制度・期限・条件
- この分野でよくある誤解・勘違い
- 専門家なら常識だが一般人は知らないこと
- 検索者が「次にぶつかる壁」（今回の疑問が解決した直後に出てくる問題）

#### 検索後の理想状態

この記事を読み終えた読者が、どんな状態になっていれば「読んでよかった」と感じるか。 知識・感情・行動の3軸で定義する。

- 知識: 何を理解している状態か
- 感情: 不安が解消されている / 自信を持てている / 判断軸ができている、など
- 行動: 何をする準備ができている状態か

### Step 3. 行動を阻害する不安・疑問

読者が「わかった、でも…」と立ち止まるポイントを洗い出す。

- 心理的ハードル（面倒くさい、失敗が怖い、恥ずかしい）
- 情報不足による迷い（自分のケースに当てはまるかわからない）
- ネガティブ情報への関心（デメリット、失敗例、落とし穴、費用）

### Step 4. 記事のゴール設計

読者のフェーズに応じて適切なゴールを1つ設定する。

| 読者フェーズ | ゴールの方向性 | 例 |
|---|---|---|
| 情報収集（まだ動く気がない） | 関連記事への遷移 / 理解の深化 | 「〇〇の具体的な手順記事に進む」 |
| 比較検討（選択肢を絞りたい） | 判断基準の提供 → 次の記事 or サービスページへ | 「自分に合う〇〇を選べる状態にする」 |
| 行動直前（背中を押してほしい） | 申込み・問い合わせ・手続き開始 | 「〇〇に申し込む」 |

※ゴールはメディアの収益モデルとも整合させる。アフィリエイトなら遷移先、リード獲得なら資料DLなど。

### Step 5. 独自性を出せる切り口（仮説）

Step 0で確認したSERPs上位記事の傾向を踏まえ、差別化できるポイントを仮説として提示する。

着眼点：

- 上位記事が「書いていない」または「浅い」情報領域
- 数字・期限・条件で具体化できるのに、上位記事が曖昧にしている部分
- 特定の読者セグメントに絞ることで深くできる領域
- 実体験・一次情報でしか得られない情報（メディアの強みに応じて）

## 注意事項

- 「なんとなく」で埋めない。SERPsの傾向と検索者心理の両方を根拠にする
- 検索意図から外れる派生ニーズは「別記事向き」と明記して除外する
- メディアのジャンル・読者層に合わせて分析の解像度を調整する（専門メディアなら知識レベルを高めに、一般向けなら平易な疑問を想定）
- KWの周辺に制度・法律・条件が絡む場合は、Web検索で現行の情報を確認してから記載する。確認できない場合は「要確認」と明記し、推測で断定しない

## 日付・情報鮮度のルール

- 分析内で年月に言及する場合は「2026年現在」を基準にする
- 「2025年現在」のように過去の日付を「現在」として記載することを禁止する
- 年号を含める場合は実際の施行年・適用年を書く。「最新」とだけ書いて年号を省略するのもNG
