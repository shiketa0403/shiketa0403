## 初月無料の適用条件（重要）

新規加入時に「加入月の基本料・視聴料が無料になる」特典は、以下の条件で**再加入時にも適用される**。

- 解約後、**1ヶ月以上の期間を空けて再加入**した場合 → 初月無料が再適用される
- 解約直後（1ヶ月未満）の再加入の場合 → 初月無料は適用されない

### 記事化時の表現

- ❌ 誤: 「再加入時は初月無料が適用されない」
- ✅ 正: 「解約から1ヶ月以上空けて再加入すれば、初月無料が再度適用される」
- ✅ 正: 「解約直後の再加入は初月無料の対象外。1ヶ月以上空ければ再適用される」

### 落とし穴

- 「すぐ再加入したい」読者には、「1ヶ月待ってから再加入すれば初月無料が使える」という具体的な提案が刺さる

## 体言止めの扱い

体言止めは「単調な語尾の連続を防ぎ、テンポを生み出すためのアクセント」として使う。使いすぎは硬い印象を生むため、配置ルールを厳守する。

### 配置ルール

- H2直下のリード文には体言止めを使わない
- 各H3配下の本文に使ってよい体言止めは最大1箇所まで（必須ではない／0箇所でも問題ない）
- H3配下の最後の文・段落では体言止めを使わない（H3の締めは必ず「です・ます」形で終わる）
- 連続使用は避ける（間に通常の文を挟む）

### H3の締めの書き方

H3配下の最後の文は、以下のいずれかで締める。

- 「〜です」「〜ます」などの通常の断定
- 「〜しましょう」などの行動喚起
- 「〜がおすすめです」などの推奨

なお、「〜したほうがよいでしょう」「〜することをおすすめします」「〜する必要があります」などの冗長表現・断定回避表現は、締めに限らず全文で禁止（上記の冗長表現・断定回避表現ルールを参照）。

### 変換例

- 「〜が条件です」→「〜が条件」
- 「〜が目安となります」→「〜が目安」
- 「〜が必要です」→「必要なのは〜」
- 「〜が向いています」→「〜に向く選択肢」

## 執筆ルール

### 1. PRE構造による深掘り（各H3の本文構造）

各H3の本文は、以下の3要素を必ずこの順序で盛り込む。

| 要素 | 役割 | 書き方 |
|---|---|---|
| P（Point） | 結論を言い切る | 冒頭1文を太字で断言する |
| R（Reason） | なぜそう言えるのか | 理由の核心を1〜2文で簡潔に示す。背景説明の厚塗りは避ける |
| E（Evidence） | 証拠で裏付ける | 具体的な数値、事例、独自情報メモの内容、公的データを提示する |

P・R・Eの各パートは1〜2文で構成する。冗長な説明で水増ししない。ただし各パートを1文で書くと論理が飛ぶ場合のみ、2文に分割してよい。

### 2. 文量の目安

| 単位 | 目安文字数 | 補足 |
|---|---|---|
| H2セクション全体 | 500〜1,000文字 | H3の数に応じて変動。密度を優先し、水増ししない |
| H3セクション | 150〜300文字 | PRE構造を各パート1〜2文で満たすと自然にこの範囲に収まる。300字を超えたら削れる表現がないか見直す |
| H2直下のリード文 | 60〜120文字 | そのH2で何がわかるかを端的に示す。構造宣言（「以下の3つを説明します」等）は禁止 |

※上記は目安であり、テーマの複雑さに応じて柔軟に調整してよい。ただし「H3が80文字で終わる」「H2が1,500文字を超える」場合は構成の分割や削減を検討する。

### 3. H2間のトランジション（つなぎ）

各H2の末尾1〜2文で、次のH2への橋渡しを行う。読者の思考の流れを途切れさせない。

| パターン | 使い方 |
|---|---|
| 因果接続 | 「〇〇がわかったところで、次に気になるのは□□です。」 |
| 疑問提起 | 「では、実際に〇〇するにはどうすればよいのでしょうか。」 |
| 視点転換 | 「ここまでは〇〇の視点でしたが、□□の側面も見ておく必要があります。」 |

毎回同じパターンを使わないこと。記事全体で2〜3パターンを混ぜる。

### 4. CTA・内部リンクの書き方

構成案に `[CTA]` が注記されているH2では、セクション末尾にCTAを自然に組み込む。

#### CTAの書き方ルール

- 読者の現在の心理状態を踏まえた導入文 → 具体的なアクション提示の順で書く
- 「詳しくはこちら」「今すぐチェック」のような汎用フレーズは禁止。読者がそのCTAをクリック/行動する理由を1文で示す
- CTA部分は本文の流れの中に溶け込ませる。唐突に出現させない

#### 出力形式

```
[CTA: （リンク先の種別）]
（CTAの導入文とアクション提示。実際のURLは空欄でよい）
```

### 5. 不安・懸念の解消

Input Dataの「不安・懸念」は、記事内の適切な箇所で必ず言及し、解消する。 解消の方法は以下のいずれかを使う。

| 方法 | 使いどころ |
|---|---|
| データで否定 | 数字や事実で不安が杞憂であることを示す |
| 条件分岐で整理 | 「〇〇の場合は心配不要、□□の場合は注意が必要」と場合分けする |
| 対処法の提示 | 不安が正当な場合、具体的な回避策・対処法を示す |

「心配ありません」「大丈夫です」だけで片付けない。根拠をセットで示す。

### 6. 具体性の徹底

- 「など」「さまざま」は禁止。具体的な名詞を3つ以上列挙する
- 数値は可能な限り明記する（例：×「費用がかかる」→ ○「初期費用で約20万円」）
- 数値の出典が明確な場合は引用ブロックで示す
- 反論・デメリットへの言及は、記事内で1〜2箇所に絞る。すべてのメリットにデメリットを併記するとトーンがネガティブに偏る

### 7. 独自情報の統合

独自情報メモの内容は、PRE構造のE（Evidence）パートで使用する。 一般論を述べた後に「実際、〇〇のケースでは」「現場の声として」等の形で自然に組み込む。 独自情報がEvidence以外の箇所で活きる場合は、そちらに配置してよい。

## 装飾ルール

| 装飾 | 用途 |
|---|---|
| 箇条書き | 3項目以上の列挙に使用 |
| 手順 | 1. 順序が重要な場合のみ |
| 表 | 比較要素が2つ以上ある場合に使用 |
| 引用 `>` | 公的データ・法令・出典を明示する場合 |

### 装飾の使いすぎ禁止

- 太字は1つのH3セクション内で最大3箇所まで
- 箇条書き・表は補助的に使用する。本文の大半が箇条書きになる構成は禁止（地の文で説明し、要点整理として箇条書きを添える）
- 引用ブロックの連続使用は禁止（引用→地の文での解説→次の引用、の流れを守る）
