# ドメイン履歴調査ツール

Wayback Machine CDX API を使い、中古ドメインの過去のタイトル変化を調査するツール。

## 用途

中古ドメイン購入前の運営履歴調査。タイトルの変化から過去の運営者数やジャンルを把握する。

## ファイル構成

| ファイル | 役割 |
|---|---|
| `domain_check.py` | 調査スクリプト本体 |
| `.github/workflows/domain_check.yml` | GitHub Actions ワークフロー |
| `csv/domains.txt` | 調査対象ドメインリスト（1行1ドメイン） |
| `csv/domain_history.csv` | 調査結果（自動生成・上書き） |

## 使い方

### 方法1: ファイルから読み込み（大量のとき）

1. GitHub **Code** → **csv** → **domains.txt** を開く
2. 鉛筆アイコンで編集 → ドメインを1行ずつ書く（`#` でコメント可）
3. **Commit changes** で保存
4. **Actions** → **ドメイン履歴調査** → **Run workflow** → **入力欄は空のまま**実行
5. 完了後 **Code** → **csv** → **domain_history.csv** で結果を確認

### 方法2: カンマ区切り入力（少数のとき）

1. **Actions** → **ドメイン履歴調査** → **Run workflow**
2. 入力欄にドメインをカンマ区切りで入力（例: `example.com,example2.com`）
3. 実行 → 結果は同じく `csv/domain_history.csv` に保存

## CSV出力フォーマット

| 列 | 説明 |
|---|---|
| ドメイン | 調査対象ドメイン（2行目以降は空欄） |
| ステータス | ok / no_archive / redirect |
| 初回アーカイブ | Wayback Machineでの最初の記録 |
| 最終アーカイブ | Wayback Machineでの最後の記録 |
| タイトル変化回数 | タイトルが変わった回数（1なら1運営者候補） |
| 時期 | タイトルが確認された年月 |
| タイトル | サイトタイトル |
| 備考 | パーキング / リダイレクト（該当時のみ） |

## 結果の見方

- **タイトル変化回数1** → 1運営者候補（購入候補）
- **備考「パーキング」** → ドメインパーキング期間あり
- **備考「リダイレクト」** → 他サイトへリダイレクトされていた期間あり
- **ステータス「no_archive」** → Wayback Machineに記録なし

## 制限事項

- Wayback Machine APIの応答はタイミングによりブレる（同じドメインでも実行ごとに取得件数が変わることがある）
- 各年2回（年始+年央）のサンプリングなので、短期間のタイトル変化は見逃す可能性あり
- 大量ドメイン（90件〜）は1〜2時間かかる。タイムアウトは180分に設定済み
- 10件ごとに途中結果を保存するため、タイムアウトしても処理済み分はCSVに残る

## 注意事項

- この環境から archive.org への直接接続は不可（GitHub Actions経由で実行する）
- `csv/domain_history.csv` は実行ごとに上書きされる（過去の結果はgit履歴から確認可能）
- WordPress自動投稿（CLAUDE.md）とは独立したツール。お互いに干渉しない
