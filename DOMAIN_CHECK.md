# ドメイン履歴調査ツール

Wayback Machine CDX API を使い、中古ドメインの最新タイトルを一括取得するツール。

## 用途

中古ドメイン購入前の調査。最新のサイトタイトルからドメインの用途を把握する。

## ファイル構成

| ファイル | 役割 |
|---|---|
| `domain_check.py` | 調査スクリプト本体 |
| `.github/workflows/domain_check.yml` | GitHub Actions ワークフロー |
| `csv/domains.txt` | 調査対象ドメインリスト（1行1ドメイン） |
| `csv/domain_history.csv` | 調査結果（自動生成・上書き） |

## 使い方

### 方法1: ファイルから読み込み（大量のとき・推奨）

1. GitHub **Code** → **csv** → **domains.txt** を開く
2. 鉛筆アイコンで編集 → ドメインを1行ずつ書く（`#` でコメント可）
3. **Commit changes** で保存
4. **Actions** → **ドメイン履歴調査** → **Run workflow** → **入力欄は空のまま**実行
5. 完了後 **Code** → **csv** → **domain_history.csv** で結果を確認

### 方法2: カンマ区切り入力（少数のとき）

1. **Actions** → **ドメイン履歴調査** → **Run workflow**
2. 入力欄にドメインをカンマ区切りで入力（例: `example.com,example2.com`）
3. 実行 → 結果は同じく `csv/domain_history.csv` に保存

## CSV出力フォーマット

| 列 | 説明 |
|---|---|
| ドメイン | 調査対象ドメイン |
| 最終アーカイブ | Wayback Machineでの最後の記録年月 |
| タイトル | 最新のサイトタイトル |
| 備考 | パーキング / リダイレクト / アーカイブなし / タイトル取得失敗 |

## 結果の見方

- **備考が空欄** → 正常に取得できたドメイン
- **備考「パーキング」** → ドメインパーキング状態（Parking Page等）
- **備考「リダイレクト」** → 他サイトへリダイレクトされている
- **備考「アーカイブなし」** → Wayback Machineに記録なし
- **備考「タイトル取得失敗」** → アーカイブはあるがタイトルを取得できなかった

## 取得失敗時の対応

タイトル取得失敗のドメインは、Wayback Machine（https://web.archive.org/）で直接確認する。

## 制限事項

- Wayback Machine APIの応答はタイミングによりブレる（同じドメインでも結果が変わることがある）
- 5ドメイン並列で処理。200件で1〜2時間程度
- タイムアウトは180分に設定済み
- 20件ごとに途中結果を保存するため、タイムアウトしても処理済み分はCSVに残る

## 注意事項

- この環境から archive.org への直接接続は不可（GitHub Actions経由で実行する）
- `csv/domain_history.csv` は実行ごとに上書きされる（過去の結果はgit履歴から確認可能）
- WordPress自動投稿（CLAUDE.md）とは独立したツール。お互いに干渉しない
